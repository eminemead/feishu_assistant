# GitLab Issue Templates

## Bug Report Template
```
Title: [BUG] Brief description of the bug

Description:
## What happened
[Describe the issue]

## Expected behavior
[What should have happened]

## Steps to reproduce
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Environment
- Project: [project name]
- Version: [version]
- Environment: [dev/staging/prod]

## Additional context
[Add any other context about the problem]
```

## Feature Request Template
```
Title: [FEATURE] Brief description of the feature

Description:
## Problem statement
[What problem does this solve?]

## Proposed solution
[Describe the solution]

## Alternatives considered
[What other approaches did you consider?]

## Acceptance criteria
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Priority
[High/Medium/Low]

## Additional context
[Add any other context]
```

## Task Template
```
Title: [TASK] Brief description of the task

Description:
## Objective
[What needs to be accomplished]

## Requirements
- [ ] Requirement 1
- [ ] Requirement 2
- [ ] Requirement 3

## Dependencies
- [ ] Dependency 1
- [ ] Dependency 2

## Due date
[YYYY-MM-DD]

## Assignee
[username]

## Additional notes
[Any additional information]
```

## Common Labels for DPA Projects
- `bug` - Bug reports
- `enhancement` - Feature requests
- `documentation` - Documentation updates
- `urgent` - Urgent issues requiring immediate attention
- `good-first-issue` - Good for newcomers
- `help-wanted` - Community help requested
- `wontfix` - Issues that won't be fixed
- `duplicate` - Duplicate issues
- `question` - User questions
- `data-pipeline` - Related to data pipelines
- `dagster` - Dagster-specific issues
- `infrastructure` - Infrastructure issues
- `security` - Security-related issues
- `performance` - Performance issues
- `testing` - Testing-related issues
